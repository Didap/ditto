# Ditto — Design Context

## Users

Designers and front-end developers who need to **understand or replicate** another
site's design system without doing it by hand. Two primary jobs:

1. **Extract** — feed a URL, get back a complete token set (colours, typography,
   spacing, radii, components) plus generated kits (CSS vars, WordPress theme,
   Elementor kit, Figma variables, React components).
2. **Inspire & blend** — feed 2–10 URLs and a "mood profile," generate a hybrid
   design system that combines the best of each reference.

Usage context: a designer/dev at a desk mid-project, moving fast, usually dark
editor or Figma already on screen. They're here to skip grunt work, not to be
entertained. Quality of the extracted tokens is the product — the interface is
the shop window for that quality.

## Brand Personality

Three words: **electric, playful, honest**.

- **Electric** — the primary is `#03e65b` (high-chroma green). Everything that
  emits signal glows. This is a confident, current tool, not a beige enterprise app.
- **Playful** — blob morphs, cursor follower, scrambling headlines, pixel stars,
  a Lottie rocket in the CTA. The product takes the work seriously; the brand
  refuses to take itself seriously.
- **Honest** — the landing copy sells what Ditto actually does, including the
  "zero AI tokens for extraction" differentiator. No promises of magic. The UI
  should feel the same: direct labels, no aspirational haze, no generic dashboard
  chrome pretending to be "AI-powered."

The interface should make someone say *"huh, someone actually designed this"* —
not *"this was generated by v0."*

## Aesthetic Direction

- **Dark-first by default**, light mode fully supported (users often switch for
  screenshots / share contexts). Theme stored in `localStorage`, flash-prevented
  in `layout.tsx` head.
- **Signature palette** (dark):
  - bg `#0a0a0a` / surface `#141414` / surface-2 `#1e1e1e`
  - primary `#03e65b` (electric green)
  - accent `#ff3386` (hot pink — used very sparingly, for *real* emphasis)
  - secondary `#ffc533` (amber — hover affordances, quest badges)
  - info `#33d0ff` (cyan — links, figma-push states)
  - text `#e5e5e5`, muted `#94a3b8`
- **Typography**: custom local fonts — `leoSans` (display) + `canvaSans` (body).
  Loaded via `next/font/local` in `src/lib/fonts.ts`. Do not introduce Inter,
  Fraunces, DM Sans, Space Grotesk, or any other reflex Google Fonts.
- **Shape language**:
  - Radii are intentionally off-grid: `6px / 8.4375px / 12.4375px` — keep this
    signature, don't "clean it up" to 8/12/16.
  - **Blobs over rectangles** when the element is emphatic (CTAs, section
    transitions, brand moments). `.ditto-blob`, `.btn-blob`, `.blob-row-top`.
  - Rectangles with 1px borders for utility surfaces (cards, form fields,
    tables). Never add 3px/4px side-stripes as decoration.
- **Motion vocabulary**:
  - Scramble text reveal on headlines (`ScrambleText`, `ScrambleWord`).
  - FontGlitch on marquee words — cycles fonts without shifting layout
    (sizer overlay pattern).
  - Blob morph on emphasis surfaces (CTA wrappers).
  - Pixel-star twinkle as ambient texture on hero.
  - Cursor follower on desktop.
  - Lottie rocket in landing CTA.
  - No bounce/elastic. Exponential ease-out on entrances.

## Anti-references

What this product is **NOT**:

- Not a Vercel-style marketing page (deep-blue-to-purple gradients, generic
  pill-shaped primaries, Inter everywhere). Ditto's palette is green/pink,
  not blue/purple.
- Not a Linear/Notion "calm and clinical" dashboard. Ditto is louder on
  purpose — a tool for people who enjoy the craft.
- Not AI-slop: no glassmorphism, no purple-to-blue gradients, no sparkline
  decoration, no border-left colour stripes, no gradient text *except* the
  single established `.ditto-gradient-text` class used sparingly on hero
  words (legacy brand element — acceptable exception).
- Not a corporate dashboard: no identical card grids, no monotonous spacing,
  no centered-everything layout.

## Design Principles

1. **Every signal colour means something.** Green = primary action / success
   pathway. Pink = critical emphasis (destructive, quest-unlock moments).
   Amber = reward / progress. Cyan = links / pushes to external tools.
   Don't decorate with them.
2. **Type does the heavy lifting.** Hierarchy comes from weight and size
   contrast in the leoSans/canvaSans pair — not from extra chrome. Aim for
   ≥1.25 ratio between heading steps.
3. **Blobs for emphasis, rectangles for work.** If an element is functional
   (input, row, card in a list), use a clean rectangle. If it's a moment
   (hero CTA, purchase button, quest unlock), let it breathe as a blob.
4. **Animate layout-safe only.** Use transform, opacity, filter. Reserve
   intrinsic sizes with sizer overlays when animating type. Never cause
   reflow in the viewport from an animation.
5. **Honest empty states.** "No designs yet — paste a URL to extract one"
   teaches the product. Not "Nothing here yet."
